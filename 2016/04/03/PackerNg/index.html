下一代Android打包工具，1000个渠道包只需要5秒

发表于 2016-04-03   |   阅读次数

简书
Github
知乎

多渠道打包是App上线前必要的一个步骤，以往的多渠道打包方式大多是通过集成友盟服务等其它方式，然而速度并不是很快，可以说数量多的时候是很慢的，今天要介绍的PackerNg打包方式，速度上简直是完胜以往的各种多渠道打包方式。
PackerNg的GitHub链接：https://github.com/mcxiaoke/packer-ng-plugin

PackerNg使用步骤

PackerNg有两种打包方式，一种是渠道打包脚本方式，另一种是命令行打包脚本方式。渠道打包脚本方式可按照GitHub上所说的步骤来，很简单。这里只说下比较酷的用命令行打包脚本方式。

命令行打包脚本方式

命令行打包脚本方式也分两种。因为可以执行java脚本来做，也可以执行Python脚本来做，这两种任选一种。首先到https://github.com/mcxiaoke/packer-ng-plugin 里把项目下载下来，项目的根目录下有markets.txt。在项目的 tools 目录里有 apkinfo.py ，ngpacker.py和ngpacker-x.x.x-capsule.jar。拷贝这些文件出来，我们都可能用得上。

markets.txt文件包含了所有我们要打包的市场信息，可根据需求自行更改，#号后面是注释。

apkinfo.py ，ngpacker.py是Python脚本文件
ngpacker-x.x.x-capsule.jar是java脚本文件

Java脚本方式

这种方式，我们需要的文件有：ngpacker-x.x.x-capsule.jar和markets.txt
